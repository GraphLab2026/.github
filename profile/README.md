# GraphRoadMap

## Historical Timeline

Graph technology has experienced rapid development from 2015 to the present, evolving from early OLTP graph databases to OLAP graph analytics, and now moving towards AI-driven lightweight and intelligent solutions.

## Related Technology Development

### Graph Databases & Graph Computing Platforms

| Project | Website | Description |
|---------|---------|-------------|
| **Neo4j** | [neo4j.com](https://neo4j.com/) | Mature graph database with Cypher query language |
| **Nebula Graph** | [nebula-graph.io](https://nebula-graph.io/) | Compute/storage separation architecture |
| **HugeGraph** | [hugegraph.apache.org](https://hugegraph.apache.org/docs/introduction/) | Apache project, AI-driven lightweight |
| **JanusGraph** | [janusgraph.org](https://janusgraph.org/) | Titan fork, distributed graph database |
| **PuppyGraph** | [puppygraph.com](https://www.puppygraph.com/) | Data lake graph engine (Iceberg/Hudi) |
| **GStore** | [gstore.cn](https://www.gstore.cn/#/) | RDF knowledge graph |
| **GraphScope** | [graphscope.io](https://graphscope.io/) | Alibaba's one-stop graph computing platform |
| **GraphScope NeuG** | [graphscope.io/neug](https://graphscope.io/neug/) | Embedded graph database, "DuckDB for Graphs" |
| **TuGraph** | [tugraph.tech](https://tugraph.tech/?lang=en-US) | Ant Group, LDBC SNB record holder |
| **TuGraph-Analyze: Geaflow** | [GitHub](https://github.com/apache/geaflow/blob/master/README_cn.md) | Unified stream-batch graph computing |

> **Observation**: Alibaba's infrastructure has strong product thinking, but when integrating with specific business scenarios, there are technical bottlenecks that cannot be overcome. In the AI era, how can this problem be solved?
